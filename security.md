# Security Policy

## Supported Versions

This project is a school website created for educational purposes.
Only the latest version in the `main` branch is supported.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes |
| Older versions | ❌ No |

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

### How to Report
- Do NOT create a public issue for security problems.
- Contact the project owner directly through GitHub.
- Clearly describe the issue and steps to reproduce it.

### What to Include
- Description of the vulnerability  
- Steps to reproduce  
- Screenshots or code snippets (if helpful)  
- Your GitHub username (optional)

## Scope

This security policy applies to:
- HTML, CSS, and JavaScript files
- Login form behavior
- Client-side validation

Since this project is **frontend-only**, no real user data is stored.

## Response Time

Security reports will be reviewed as soon as possible.
Fixes will be applied based on severity and school project requirements.

## Disclaimer

This project is for **school and educational use only**.
The login system is a demo and not intended for real authentication.

---

Thank you for helping keep this project safe!
